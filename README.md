# 🤖 ربات تلگرام ردیاب موجودی

لینک محصول میفرستی، سایز انتخاب می‌کنی، وقتی موجود شد پیام میاد.

---

## ساختار فایل‌ها

```
restock-bot/
├── bot.py           ← ربات تلگرام (برای تست لوکال)
├── checker.py       ← چکر خودکار (GitHub Actions اجرا میکنه)
├── watchlist.json   ← لیست محصولات (خودکار آپدیت میشه)
├── requirements.txt
└── .github/
    └── workflows/
        └── check.yml
```

---

## راه‌اندازی (یه‌بار)

### ۱. ساخت ربات تلگرام

به **@BotFather** پیام بده:
```
/newbot
```
اسم و username بده، توکن بگیر.

### ۲. ساخت GitHub Repo

یه repo جدید بساز (می‌تونه خصوصی باشه)، همه فایل‌ها رو آپلود کن.

### ۳. اضافه کردن Secret

برو به **Settings → Secrets and variables → Actions → New repository secret**:

| نام | مقدار |
|-----|-------|
| `TELEGRAM_BOT_TOKEN` | توکنی که از BotFather گرفتی |

> نیازی به CHAT_ID نیست — ربات chat_id هر کاربر رو خودش از پیامشون میگیره!

### ۴. اجرای ربات

**روش A — تست لوکال (روی کامپیوتر خودت):**
```bash
pip install requests beautifulsoup4
export TELEGRAM_BOT_TOKEN="توکنت"
python bot.py
```

**روش B — سرور دائمی (توصیه‌شده برای استفاده واقعی):**

از یه VPS ارزون یا Railway.app یا Render.com استفاده کن:
```bash
python bot.py
```

**روش C — فقط چکر (بدون bot polling):**

اگه فقط می‌خوای GitHub Actions چک کنه و bot.py رو دستی اجرا کنی برای اضافه کردن، همین کافیه. چکر به‌صورت خودکار هر ۳۰ دقیقه یه‌بار کار می‌کنه.

## دستورهای ربات

| دستور | کار |
|-------|-----|
| `/start` | خوشامدگویی |
| `/list` | لیست محصولات دنبال‌شده |
| `/clear` | پاک کردن همه |
| ارسال لینک | شروع فرایند اضافه کردن |

## نکات

- `watchlist.json` خودکار توی GitHub ذخیره میشه (هر بار چک می‌کنه commit میده)
- می‌تونی چند کاربر داشته باشی — هر کاربر لیست جداگانه داره
- اگه سایتی anti-bot داشت و scrape نشد، issue باز کن
